# Sizing, Not Signaling

An option-implied cross-asset allocation rule.

**Author:** Pavlos Giannakis  
**Course:** FINM 33150 — Quantitative Trading Strategies, Spring 2026, University of Chicago  
**Due:** 25 May 2026, 11:59 PM CST

## What this project does

The project tests whether option-implied volatility can be useful for sizing risk
in a cross-asset ETF portfolio. The strategy trades eight liquid US-listed ETFs
(SPY, IWM, EFA, EEM, XLK, XLF, GLD, TLT) and does **not** try to predict which
asset will outperform next. Instead:

1. **Sleeve sizing.** Each ETF is sized inversely to its 30-day ATM implied
   volatility (from WRDS OptionMetrics), so higher-vol assets get smaller
   notional weights and vice versa.
2. **Gross exposure gate.** A four-state macro regime built from VIX and the
   Chicago Fed NFCI scales the total gross exposure of the book (calm 1.5×,
   normal 1.0×, stress 0.5×, crisis 0.0×).

All free parameters are tuned on a 2008–2017 design window and frozen before
the 2018–2024 evaluation window is touched.

Headline result on USD 1.00M of starting capital:

| Strategy | Terminal wealth | Max drawdown | Sharpe |
|---|---:|---:|---:|
| IV-sized, regime-gated | USD 1.48M | -14.5% | +0.40 |
| Equal-weight 8-ETF | USD 2.64M | -45.2% | +0.42 |

The strategy gives up terminal wealth in exchange for a smoother path and a
roughly 3× smaller worst drawdown. In-sample Sharpe +0.47 → held-out Sharpe
+0.28 (performance decays but stays positive).

## Running the notebook

### Option A — just open the executed notebook

The committed `.ipynb` already has all 20 figures rendered. Open it in
JupyterLab, classic Jupyter, or VS Code's notebook viewer and read top to
bottom. No environment setup needed.

### Option B — re-execute end-to-end

On macOS or Linux:

```bash
git clone https://github.com/paulgiann/finm33150-final-project.git
cd finm33150-final-project
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install jupyterlab
jupyter lab final_project_notebook_v3.ipynb
```

On Windows (PowerShell):

```powershell
git clone https://github.com/paulgiann/finm33150-final-project.git
cd finm33150-final-project
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
pip install jupyterlab
jupyter lab final_project_notebook_v3.ipynb
```

The setup cell locates `notebook_data/` automatically. It first checks the
`FINM_PROJECT_DATA` environment variable, then looks for `notebook_data/`
beside the notebook, then walks up parent directories. If the data folder is in a non-standard location, point the env var at it.

macOS/Linux:

```bash
export FINM_PROJECT_DATA=/path/to/notebook_data
```

Windows PowerShell:

```powershell
$env:FINM_PROJECT_DATA = "C:\path\to\notebook_data"
```

### Option C — rebuild the notebook from source

The committed `.ipynb` is generated by `build_notebook_final.py`. Regenerate
the (unexecuted) notebook with:

```bash
python build_notebook_final.py
```

Then execute it programmatically:

```bash
python -c "import nbformat; from nbclient import NotebookClient; nb = nbformat.read('final_project_notebook_v3.ipynb', as_version=4); NotebookClient(nb, timeout=300, kernel_name='python3').execute(); nbformat.write(nb, 'final_project_notebook_v3.ipynb')"
```

## Data provenance

The parquet files in `notebook_data/` are derived from:

- **WRDS OptionMetrics** — constant-maturity / constant-delta implied
  volatility surfaces at 30/60/91-day tenors and 25Δ-call / ATM / 25Δ-put
  strikes for the eight ETFs.
- **CBOE VIX** and **Chicago Fed NFCI** — daily macro stress inputs.
- **Yahoo Finance / CRSP** — daily spot adjusted-close prices for the eight
  ETFs.

The raw WRDS pull and the strategy backtest are not included in this repo
because (a) WRDS access is licensed, and (b) the backtest is already
embedded in the committed parquet files. `notebook_data/AUDIT.md` documents
which columns flow into which figure.
